## Questions

1. Explain what a Python module is and what problem it solves. Describe a real project (such as a web app, a data processing tool, or a game) and explain how you would split that project into multiple modules. What criteria would you use to decide what goes in each module?

2. The `if __name__ == "__main__":` guard is a Python convention used in almost every module. Explain exactly what it does, why it is necessary, and describe what would happen in a real project if you forgot to include it in a module that contained test code or print statements at the top level.

3. Virtual environments solve a specific problem in Python development. Describe that problem in detail — what goes wrong without virtual environments when you work on multiple projects? Then explain how a virtual environment solves it and why `requirements.txt` is an essential companion to the virtual environment.

4. Compare `import module` with `from module import x`. Describe the trade-offs of each approach in terms of readability, namespace pollution, and maintainability. When would you choose one over the other in a real team project?
