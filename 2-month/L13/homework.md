## Questions

1. `*args` and `**kwargs` both allow functions to accept a variable number of inputs, but they work differently. Explain the difference between the two, including how the arguments are stored inside the function. Give a real example of a function where `*args` is the right choice and a different function where `**kwargs` is the right choice.

2. Mutable default arguments are one of Python's most famous traps. Explain exactly what the problem is with `def append_item(item, lst=[])`, show step-by-step what happens when this function is called multiple times, and explain the correct way to write it. Why did this design decision happen in Python?

3. Lambda functions are often described as "anonymous functions." Explain what this means, describe when it is appropriate to use a lambda, and describe when you should use a regular `def` function instead. Give a real example of each case.

4. Default parameters allow a function to be called with fewer arguments than it defines. Explain how this feature improves the usability of a function. Give an example of a real function (from a library, an API, or your own experience) where defaults make the function much easier to use in simple cases while still being flexible for advanced use.
