## Questions

1. In today's project, you used a generator for ID creation, a closure for filtering, and a decorator for logging — all solving different problems with functions as the tool. Looking back, explain why each of these three patterns was the right choice for its specific role. Could you have solved any of them with a simpler approach? What would you have lost by doing so?

2. The project was built entirely with functions — no classes. Describe the advantages and limitations of this approach. At what point in the project's complexity would you feel compelled to switch to classes, and why?

3. Using comprehensions for all filtering and transformation kept the code concise. Identify two places in your project where a comprehension replaced what would have been a longer `for` loop with an `if` inside. For each, explain whether the comprehension improved readability or made it harder to understand.

4. JSON was used for persistence in this project. What are the limitations of JSON as a storage format for a task manager? If this were a real production tool used by hundreds of people, what storage system would you replace it with and why?
