## Questions

1. A closure is a function that "remembers" variables from its enclosing scope. Explain in your own words why this is useful — what problem does it solve that you cannot solve just by passing arguments normally? Use a real-world analogy (not a programming one) to explain how a closure "remembers" something.

2. `map()`, `filter()`, and `reduce()` are called higher-order functions because they take functions as arguments. Explain what a higher-order function is, and describe a real data processing scenario (such as processing a list of orders, employees, or sensor readings) where using `map()` and `filter()` together would make the code cleaner than using a loop.

3. Compare using a `for` loop versus using `map()` or `filter()` to process a list. What are the advantages and disadvantages of each approach? When would you choose one over the other in a professional codebase?

4. `reduce()` is powerful but can be confusing to read, which is why it was moved to `functools` and is used less often in modern Python. Explain what `reduce()` does step-by-step with a concrete example, and describe a situation where it is genuinely the most elegant solution — and one where it is overkill.
