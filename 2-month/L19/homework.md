## Questions

1. Regular expressions are described as a "mini programming language" inside a string. Explain why learning regex is worth the initial difficulty. Describe three real situations in backend development — such as input validation, log parsing, or data extraction — where regex is the best tool for the job and a simple `if` statement would be insufficient.

2. `re.search()` and `re.match()` both look for a pattern in a string but behave differently. Explain precisely what each one does, how they differ, and give a concrete example of when you would specifically choose one over the other. What mistake could occur if you used the wrong one?

3. JSON uses `null`, `true`, and `false` while Python uses `None`, `True`, and `False`. Explain what happens to these values during JSON serialization (`json.dumps`) and deserialization (`json.loads`). Why is it important to know about this conversion when working with APIs?

4. Compare `json.dumps()` vs `json.dump()` and `json.loads()` vs `json.load()`. Many developers confuse these four functions. Explain what each one does, when you would use each one, and describe a real scenario where confusing `dump` with `dumps` could cause a silent bug or a crash.
