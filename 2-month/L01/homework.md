## Questions

1. A function is often described as a "black box" — you put something in and get something out, without needing to know what happens inside. Explain why this idea is valuable when building large software. How does this principle help a team of developers work together on the same project?

2. Explain the difference between a **parameter** and an **argument**. Then explain the difference between a **local variable** and a **global variable**. Use a real-world analogy (not a programming one) to make the scope concept clear to someone who has never programmed before.

3. The `global` keyword allows a function to modify a variable defined outside it. However, experienced developers avoid using `global` whenever possible. Explain why overusing `global` is considered bad practice. What problems can it cause in a program with many functions?

4. Some functions `return` a value; others only perform an action and return `None`. Describe two real situations where a function should return a value, and two situations where returning `None` (action only) is the correct design. How do you decide which one to use?
