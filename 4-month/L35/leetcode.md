## Problem 1 — Big Countries

- **Link:** https://leetcode.com/problems/big-countries/
- **Difficulty:** 🟢 Easy
- **Why this lesson:** Just like branching in Git lets you work on multiple things in parallel, the `OR` operator in SQL lets you select rows matching multiple different conditions simultaneously. Practice combining filter conditions.
- **Hint:** A country is "big" if it meets at least one of two criteria — think about which SQL operator means "at least one must be true."
- **Your file:** `leetcode_1.sql`

## Problem 2 — Article Views I

- **Link:** https://leetcode.com/problems/article-views-i/
- **Difficulty:** 🟢 Easy
- **Why this lesson:** This problem introduces filtering rows where two columns hold the same value — a pattern you will use constantly in PostgreSQL JOINs next lesson. It also introduces `DISTINCT` and `ORDER BY`, which are essential for clean query results.
- **Hint:** You need to find rows where the author and the viewer are the same person. Think about how to compare two columns in a `WHERE` clause.
- **Your file:** `leetcode_2.sql`
