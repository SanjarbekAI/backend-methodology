## Questions

1. Explain why `UPDATE` and `DELETE` without a `WHERE` clause are among the most dangerous SQL commands. Describe the professional workflow a developer should follow before running any `UPDATE` or `DELETE` in production — step by step. What does `RETURNING` help you verify?

2. Compare `ON DELETE CASCADE`, `ON DELETE RESTRICT`, and `ON DELETE SET NULL`. For each one, describe a specific real-world relationship between two tables where that behavior is the **correct** choice, and explain what would go wrong if you chose one of the other options instead.

3. Explain what a database index is, using a real-world analogy. Describe the trade-off: what does an index give you, and what does it cost? Why is it a mistake to add an index on every column of every table "just to be safe"?

4. What is a database migration, and why should schema changes be tracked as versioned migration files instead of just running `ALTER TABLE` statements directly on the database? Describe what problems arise on a team of 4 developers if nobody uses migrations.
