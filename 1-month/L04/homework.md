## Questions

1. String slicing uses the pattern `[start:stop:step]`. Explain in your own words what each of the three parts does, and describe a real situation — such as processing a product code, a phone number, or a date string — where slicing would be genuinely useful instead of just reading the whole string.

2. Python string methods like `upper()`, `strip()`, and `replace()` do not modify the original string — they return a new one. Why do you think Python was designed this way? What could go wrong in a program if string methods modified the original string directly?

3. Compare f-strings with the older `.format()` method. For example: `f"Hello {name}"` versus `"Hello {}".format(name)`. Both produce the same result. Which would you use and why? What makes f-strings easier to read and maintain in a large codebase?

4. A company's customer database stores first names and last names in inconsistent formats — some are all uppercase, some are all lowercase, and some have extra spaces. Describe step by step how you would use Python string methods to clean and standardize every name in the database. Which methods would you use and in what order?
