## Questions

1. `input()` in Python always returns a string, even when the user types a number. Why do you think Python was designed this way instead of automatically detecting the type? What are the advantages of this design, and what responsibility does it put on the developer?

2. Describe what would happen in a real application — such as a bank transfer form or an online store checkout — if there were no error handling at all. Walk through a specific scenario where a user makes a mistake (such as typing a letter instead of a number), and describe what the user would experience without `try/except` versus with it.

3. Compare the `except` clause and the `finally` clause. Explain the purpose of each, describe when each one runs, and give a real example of a situation where `finally` is essential — not just optional.

4. When a Python program raises an unhandled exception, it prints a traceback. Describe what information a traceback contains, why it is useful for a developer, and explain why showing a raw traceback to an end user (instead of a friendly error message) is bad practice in a real application.
