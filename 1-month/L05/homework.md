## Questions

1. Explain the difference between using multiple `if` statements versus using `if / elif / else`. Give a concrete example of a system (such as a grading system, a pricing tier, or an age check) where using `elif` is the correct choice, and explain what would go wrong if you used separate `if` statements instead.

2. Python considers some values as "falsy" even though they are not literally `False`. Explain what "truthiness" and "falsiness" mean in Python, list at least four falsy values, and describe a real-world scenario in a program where this behavior could cause a subtle bug if the developer was not aware of it.

3. Compare `if/elif/else` chains with the `match` statement introduced in Python 3.10. When is `match` the better choice? Are there situations where `if/elif` is still more appropriate? Use a real example to support your answer.

4. Nested conditions can solve many problems, but experienced developers try to avoid deep nesting. Explain why deeply nested code is considered bad practice, and describe what a developer can do to simplify it. Give an example of a nested condition and explain how it could be rewritten more cleanly.
