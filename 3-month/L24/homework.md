## Questions

1. Inheritance promotes code reuse by letting child classes build on parent classes. However, experienced developers often say "prefer composition over inheritance." Explain what both inheritance and composition mean, describe a situation where inheritance is the right choice, and a different situation where composition is better. Why is overusing inheritance considered a design risk?

2. `super()` is used to call a parent class's methods from a child class. Explain why this is important in `__init__` — what exactly happens if you override `__init__` in a child class without calling `super().__init__()`? Give a concrete example of the bug this would cause.

3. Polymorphism allows different objects to be used interchangeably when they share a common interface. Explain how Python's duck typing achieves this without requiring classes to share a common parent. Give a real example from software development — such as a plugin system, a payment gateway, or a report exporter — where polymorphism makes the system more flexible and extensible.

4. Multiple inheritance in Python allows a class to inherit from more than one parent. Describe the "diamond problem" — what it is, why it happens, and how Python's MRO (Method Resolution Order) resolves it. When is multiple inheritance genuinely useful (hint: think about mixins)?
