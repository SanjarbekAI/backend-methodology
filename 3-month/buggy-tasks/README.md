# Month 3 — Buggy Tasks

These tasks contain **intentionally broken Python code**. Your job is to find every bug, fix it, and make the program run correctly.

---

## When to do these

| Tasks | Give after |
|---|---|
| `bug_01.py` `bug_02.py` `bug_03.py` | After **L05** — you have covered OOP (classes, inheritance, polymorphism, encapsulation, magic methods, abstract classes, SOLID & DRY) |
| `bug_04.py` `bug_05.py` `bug_06.py` | After **L09** — you have covered concurrency (threading, asyncio), context managers, metaclasses, performance & memory |

---

## Rules

- Do **not** rewrite the program from scratch — fix only what is broken
- Every bug has a comment `# BUG` next to it — but the comment does NOT tell you what the bug is
- Some tasks have **multiple bugs** — find all of them
- Write the number of bugs you found as a comment at the top of your fixed file
- Test your fix: the program must run without errors and produce the expected output

---

## How to submit

Copy the buggy file, rename it `bug_01_fixed.py`, fix the bugs inside it, and submit the fixed version.
