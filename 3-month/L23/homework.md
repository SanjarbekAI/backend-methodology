## Questions

1. Object-Oriented Programming organizes code around objects that combine data and behavior. Compare this to the procedural style (functions and data separately) you used in Months 1 and 2. Describe a real system — such as a hospital patient tracker, a hotel booking system, or an online store — and explain which parts of that system would be modeled well as objects and why.

2. Explain the difference between an instance variable and a class variable. Give a real example of a class where both are used, and explain why each piece of data belongs at the level (instance vs class) you chose. What could go wrong if you accidentally used a mutable class variable when you needed an instance variable?

3. `self` is the first parameter of every instance method. Explain what `self` actually refers to at runtime, why Python requires you to write it explicitly (unlike languages like Java or C++ where it is implicit), and what would happen if two different instances called the same method with their own data.

4. The single underscore prefix `_attribute` is described as "private by convention." Explain what this means — what does it tell other developers, and does Python actually enforce any restriction? Compare this to the double underscore `__attribute` (name mangling). Why might a team choose one over the other?
